🗄️ 1. Database Systems
Q1. Hospital records in separate Excel files 5 Marks

(a) Two problems from redundancy & inconsistency (2 marks)

  • Wasted storage & update overhead: same patient/doctor details stored in multiple files → repeated updates, wasted space.
  • Data inconsistency: address updated in billing but not records → conflicting values → billing errors & confusion.

(b) DDL vs DML (2 marks)

  • DDL (Data Definition Language) defines structure: CREATE TABLE Patients (...)
  • DML (Data Manipulation Language) manages data: INSERT INTO Patients VALUES (...)

(c) SQL: patients older than 50 (1 mark)

SELECT * FROM Patients WHERE Age > 50;
Q2. Retail store spreadsheets 5 Marks

(a) Two limitations of file‑processing (2 marks)

  • Data redundancy & inconsistency across inventory/sales.
  • No integrity or concurrent access control (e.g. stock negative, simultaneous updates).

(b) DDL & DML in RDBMS (2 marks)

  • DDL defines tables (customers, products, transactions).
  • DML manipulates data (insert sales, update stock, delete products).

(c) SQL: products priced above 1000 (1 mark)

SELECT * FROM Products WHERE Price > 1000;
Q3–Q6. College, Hospital, Company scenarios selected highlights

Redundancy effect on decision‑making: conflicting values across files → mistrust, poor decisions (e.g. fee defaults).

DDL vs DML examples: CREATE TABLE ExamResults (...) vs UPDATE ExamResults SET GPA=3.5 WHERE ID=10;

SQL queries: SELECT * FROM Students WHERE Program='BBA'; · SELECT * FROM Patients WHERE Age>60; · SELECT * FROM Employees WHERE Salary>50000;

CREATE vs INSERT: CREATE defines structure (DDL); INSERT adds rows (DML).

☁️ 4. Cloud Computing
Q1. Startup online shopping 5 Marks

(a) Three advantages: cost savings (pay‑as‑you‑go), scalability (handle spikes), accessibility (remote mgmt).

(b) IaaS / PaaS / SaaS:

  • IaaS – Amazon EC2 (virtual infrastructure)
  • PaaS – Google App Engine (managed platform)
  • SaaS – Microsoft 365 (ready‑to‑use app)
Q2–Q6. Education, Accounting, Migration, etc.

Benefits: scalability, accessibility, cost reduction, automatic updates, disaster recovery.

Examples: AWS EC2 (IaaS), Google App Engine (PaaS), QuickBooks Online / Salesforce (SaaS).

Migration: converts capex to opex, elastic scaling, no physical infrastructure.

🛡️ 5. Virtualization & Cybersecurity
Q1. University lab & CIA Triad 5 Marks

(a) Virtualization: single physical server → multiple VMs via hypervisor (Windows + Linux on same hardware).

(b) CIA Triad examples:

  • Confidentiality: encrypt exam results
  • Integrity: checksums to detect tampering
  • Availability: backup servers for uptime
Q2–Q5. VM benefits, hypervisor, security controls

VM benefits: clean disposable test environments, isolated OS, quick reset.

Hypervisor: allocates CPU/memory/storage, keeps VMs isolated.

CIA controls: ACLs/encryption (confidentiality), hashing/audit (integrity), redundancy/backups (availability).

🥽 6. AR & VR
Q1. Interior design app 5 Marks

(a) Differences: AR overlays digital content on real world (phone); VR fully replaces with simulated environment (headset).
AR more suitable for interior design (see furniture in actual room).

(b) Applications: AR: IKEA Place, navigation, anatomy; VR: gaming, flight sims, virtual tourism.

Q2–Q6. Museum, navigation, healthcare, education

VR for virtual museum – fully immersive experience.
AR for navigation – overlays directions on live view.
Healthcare/Education: AR (3D anatomy, surgical overlay), VR (simulation training, virtual classrooms).

📡 7. Internet of Things (IoT)
Q1. Smart Healthcare 5 Marks

4‑layer mapping:

  • Perception: wearable sensors (vital signs)
  • Network: Wi‑Fi / Bluetooth transmission
  • Processing: cloud/edge analysis
  • Application: doctor dashboard / alert app

Data flow: sensors → network → processing (abnormality detection) → alert → doctor app.

Q2–Q6. Agriculture, Parking, Smart Home

Agriculture: soil sensors (Perception) → LoRa/Wi‑Fi (Network) → server analysis (Processing) → farmer app (Application).

Smart Parking: occupancy sensors → wireless → server maps vacancies → driver app with guidance.

Smart Home: light/temp sensors → home Wi‑Fi → smart hub → mobile app for remote control.
